Написать приложение на Java Springboot с пользовательской логикой, например мини банк.
регистрация (ФИО, данные, телефон) - номер аккаунт, счет, карту.
авторизация по номеру телефона
переводы клиентам внутренним внешним
платежи по шаблону
Каждый из перечисленных операций делался минимум в 2-3 этапа и связаны между собой sessionid или transactionid.

Прикрутить мониторинг к БД и приложения (actuator). Скидывать метрики в Prometheus, графики рисовать в Grafna (дашборды взять стандартные на сайте графины) Вывести метрики утилизация ресурсов приклада.

Развернуть все это в Docker (не docker desktop).

Нагрузить: Для отладки сделать Postman коллекцию со операциями своего приложения Взять любой инструмент Jmeter или Gatling написать скрипты нагрузки и провести 2 теста поиск максимума и подтверждения.  К инструменту прикрутить так же стандартный мониторинг influxdb либо prometheus и дашборды в grafana.
